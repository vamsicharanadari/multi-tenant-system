2. Authentication, JWT Strategy, & Context Resolution
2.1 JWT Payload Specification
Authentication is stateless. Upon validation, the backend issues an RS256 or HS256 signed JWT.
JSON
{  "sub": "usr_9b1deb4d-3b7d-4bad-9bdd-2b0d7b3dcb6d",  "tenant_id": "tnt_110ec48a-a070-4ac8-a146-17b20a9ae369",  "role": "Admin",  "is_super_admin": false,  "impersonated_by": null,  "exp": 1784580000,  "iat": 1784544000}
2.2 FastAPI Context Resolution Flow
Request Interception: Middleware extracts Authorization: Bearer <JWT>.
Token Verification: Validates signature and expiration.
Tenant Routing: Reads tenant_id from claims. Retrieves connection URI from cache/Control DB.
Session Injection: Instantiates a scoped async SQLAlchemy session (AsyncSession) targeting tenant_<tenant_id> and injects it into the request scope (request.state.db).

5. Functional Requirements & Feature Specifications
5.1 Super Admin "God-Mode" Impersonation
Endpoint: POST /api/v1/super-admin/impersonate
Payload: { "target_tenant_id": "UUID", "target_user_id": "UUID" }
Behavior:
Validates requesting token contains is_super_admin: true.
Generates a temporary JWT (valid for 15 minutes) bound to target tenant_id and target_user_id.
Sets claim impersonated_by: <super_admin_id>.
All downstream operations audit log the action under the Super Admin's ID.
5.2 Frontend White-Labeling Engine
Execution:
On login or session refresh, React fetches GET /api/v1/tenant/config.
Response returns branding_config JSON.
React dynamically updates root CSS variables:JavaScriptdocument.documentElement.style.setProperty('--primary-color', config.primary_color);document.documentElement.style.setProperty('--secondary-color', config.secondary_color);
Dynamically switches favicon and application logo in navbar.

6. Migration Strategy (Alembic)
Migrations must handle both database types cleanly:
Control DB Migrations:alembic -c alembic_control.ini upgrade head
Tenant DB Migrations (Multi-Tenant Loop):FastAPI CLI script iterates through active tenant connection strings in control_plane_db and executes Alembic programmatically against each tenant DB:Pythonfor tenant in active_tenants:    run_async_migrations(tenant.db_connection_uri)
7. Non-Functional Requirements (NFRs)
Data Isolation Security: Zero cross-tenant SQL queries. Every request MUST obtain its DB engine dynamic binding via request.state.tenant_id.
Performance: Connection pools for individual tenant DBs must be cached using a dynamic engine factory dictionary to prevent pool creation overhead per HTTP request.
Audit Trails: Impersonation and user role updates must emit structured JSON log entries containing timestamp, actor ID, target tenant, and action.
