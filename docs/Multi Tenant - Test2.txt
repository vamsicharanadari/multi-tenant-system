# MASTER AGENT DIRECTIVE: TDD Multi-Tenant Framework Implementation
**Target Repository:** https://github.com/vamsicharanadari/multi-tenant-system
**Tech Stack:** FastAPI (Python), React (TypeScript), PostgreSQL (Async SQLAlchemy 2.0 + Alembic), Redis, Docker
**Core Methodology:** STRICT TEST-DRIVEN DEVELOPMENT (TDD).
---
## 🛑 MANDATORY TDD RULE FOR AGENT
FOR EVERY FEATURE BLOCK BELOW:
1. WRITE THE FAILING TEST FIRST IN `backend/tests/`.
2. EXECUTE `pytest` AND CONFIRM THE TEST FAILS (RED).
3. WRITE ONLY THE MINIMAL PRODUCTION CODE REQUIRED TO PASS THE TEST (GREEN).
4. REFACTOR CODE FOR CLEAN ARCHITECTURE AND RE-RUN `pytest` (REFACTOR).
DO NOT WRITE PRODUCTION IMPLEMENTATION CODE BEFORE WRITING A FAILING TEST.
---
## STEP-BY-STEP TDD EXECUTION DIRECTIVE
### BLOCK 1: Infrastructure & Environment Setup
1. Scaffold repository file layout:
├── backend/ │ ├── app/ │ │ ├── api/ │ │ ├── core/ │ │ ├── middleware/ │ │ ├── models/ │ │ └── services/ │ └── tests/ ├── frontend/ │ └── src/ │ ├── components/ │ ├── context/ │ └── pages/ └── docker-compose.yml

### BLOCK 2: TDD Auth & Dynamic Tenant Session Routing

* **Run:** `cd backend && pytest tests/test_auth_and_context.py` $\rightarrow$ **MUST PASS**.

### BLOCK 3: TDD Multi-Tenant Data Isolation
#### Step 3.1: Write Failing Tests (Red Phase)

- `test_cross_tenant_lookup_denied()`: Attempt cross-tenant ID query $\rightarrow$ assert 404 or 403.
* **Run:** `cd backend && pytest tests/test_tenant_isolation.py` $\rightarrow$ **MUST FAIL**.
#### Step 3.2: Implement Production Code (Green Phase)
* Build tenant DB base schemas (`backend/app/models/tenant.py` - `User` model bound to tenant DB scope).

* **Run:** `cd backend && pytest tests/test_tenant_isolation.py` $\rightarrow$ **MUST PASS**.

### BLOCK 4: TDD Provisioning, Impersonation & Feature Gating
#### Step 4.1: Write Failing Tests (Red Phase)
* Create `backend/tests/test_provisioning_and_gating.py`:

- `test_impersonation_generates_audited_jwt()`: Calls `POST /api/v1/super-admin/impersonate` $\rightarrow$ verifies `impersonated_by` claim in token.

* **Run:** `cd backend && pytest tests/test_provisioning_and_gating.py` $\rightarrow$ **MUST PASS**.
